# QA 抓蟲實戰：找得到、釘得死、追得到底

> 定位：抓蟲是科學，不是運氣。測試工程師最核心的硬實力 = 把 bug 找出來（破壞性思維）、釘死（重現的科學方法）、追到修好（協調與追蹤）。
> 教學哲學：每篇 hook 開場、有方法、有範例、收尾「帶得走」。不寫成 tips 清單。
> 視角：QA IC 講師。順著一隻 bug 的一生編排——找 → 釘 → 關。
> 狀態：草稿（本地）。一課一檔 `article-Axx/Bxx/Cxx-*.md`。

## 課程定位

- **破壞性測試是高階能力**：不只確認「它能動」，更要證明「它會壞」。
- **重現是可拆解的科學方法**：控制變數、邊界值、最小重現，不靠運氣。
- **缺陷追蹤是 QA 含金量最高的軟實力**：把 bug 從發現追到關閉。
- **與溝通課、品質課互補**：那兩課從溝通／管理視角，本課從 craft／方法視角。

## 課綱（9 課 / 3 模組）

### 模組 A · 破壞性思維（怎麼找到 bug）
- **A01** 逆向思維：從「它能動」到「我要弄壞它」
- **A02** 攻擊維度地圖：系統有哪些面可以打
- **A03** 怎麼認得「這是 bug」：一致性判準

### 模組 B · 重現與報告（怎麼釘死 bug）
- **B04** 重現是科學方法：控制變數與最小重現
- **B05** 偶發 bug 怎麼抓：重現率、日誌、變數隔離
- **B06** 寫一份 RD 一分鐘看懂的缺陷報告

### 模組 C · 協調與追蹤（怎麼讓 bug 被修）
- **C07** 釐清異常：把 bug 交對人
- **C08** 缺陷分級：Severity 不等於 Priority
- **C09** 驗證修復與回歸：別只測那一個點
