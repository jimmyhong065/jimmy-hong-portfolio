# L12 執行紀律：基線、單變數、暖機、可重現

**模組**：M4 執行與監控　|　**狀態**：草稿

## 學習目標

- 建立可信、可比較、可重現的壓測執行紀律。
- 知道基線（baseline）的意義與建立方式。
- 養成「單次只改一個變數 + 完整記錄」的習慣。

## 大綱

1. 開場：不可重現的壓測 = 不可信的結論。
2. 基線：第一份可信數據作為一切比較的錨。
3. 單變數原則：歸因的前提。
4. 暖機與穩態取樣。
5. 完整記錄：環境快照 + 參數 + 結果三件套。
6. 收尾：每輪壓測 SOP。

## 核心觀念

- **基線 baseline**：在已知良好配置下跑一輪、記錄完整指標，作為後續每次優化/回歸的比較錨點。沒有基線就無法說「變快/變慢」。
- **單變數原則**：一次只改一處（配置、程式、資料、流量其一），否則效能變化無法歸因。
- **暖機 + 穩態**：排除 ramp-up / JIT / 快取填充段，只取穩態窗口的指標（見 L09）。
- **可重現三件套**：
  1. 環境快照：版本、機器規格、配置、資料量。
  2. 測試參數：流量模型、加壓曲線、腳本版本、think time。
  3. 結果：完整指標 + 監控截圖 + 原始數據存檔。
- 多輪取平均/中位，避免單次抖動下結論；注意環境的其他干擾（旁路流量、排程）。

## 實戰／案例

- 翻車：「上次測比較快」但沒人記得上次的配置/資料量/版本，爭論無解——基線與記錄缺失。
- 同時改了 JVM 參數又加了索引，效能變好，但不知道是哪個生效，無法複製到生產。
- SOP：每輪壓測產出一份固定模板（環境/參數/結果/結論/下一步假設），存版控或 wiki，可追溯。

## 書本參考

《性能測試實戰30講》ch06（場景的比對）：高樓強調只改單一因素、其他環境/數據/配置不變才能歸因，且固定條件下系統最大 TPS 上限是固定的——本課的基線與單變數原則即此準則的執行版。

## 內鏈

- L06（環境）— 環境快照與變數隔離。
- L09（加壓）— 暖機排除、穩態取樣。
- L13（排查）— 單變數是定位歸因的前提。
- L19（報告）/L20（回歸）— 記錄三件套是報告與回歸的素材。
