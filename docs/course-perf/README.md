# 效能測試實戰課程 — 筆記草稿

> 主題：大型電商活動（雙11、限時秒殺）高併發壓力測試 — 規劃、執行、系統與資料庫瓶頸定位。
> 視角：QA IC 講師，帶讀者走完一整套效能測試方法論。
> 狀態：草稿（本地）。每課兩檔——`Lxx-*.md`（筆記骨架）+ `article-Lxx-*.md`（已展開的完整文章，~1400 字，含 frontmatter / hook / 案例 / 收尾，部分含 Mermaid 圖）。20 篇文章全數完成，待發布。

## 參考書

**主要來源**：《性能測試實戰30講》高樓著（極客時間，32 篇）。實際讀取全文抽取，引用可考證。

- 本課程引用其核心立場：TPS 為關鍵指標（非線程數）、TPS 判容量/RT 判快慢、不追拐點看趨勢、分析決策樹與證據鏈、只測不調 = 性能驗證。
- 也用它破除迷信：28 原則、258/2510 響應時間原則、並發公式 C=nL/T、線程數=用戶數、light/heavy/buckle=性能/負載/壓力測試。

## 課綱（6 模組 / 20 課）

### M1 基礎心法
- L01 釐清術語：效能 / 負載 / 壓力 / 浸泡測試差別
- L02 為何雙11／秒殺難測：尖峰、長尾、流量模型
- L03 指標體系：RT、P95/P99、TPS/QPS、吞吐、併發數、錯誤率

### M2 需求與規劃
- L04 從業務目標推效能目標（SLA/SLO、預估 QPS 怎麼算）
- L05 建流量模型：用戶旅程、讀寫比、熱點數據
- L06 環境與造數：環境等比、測試資料、變數隔離

### M3 工具與腳本
- L07 工具選型：JMeter / k6 / Gatling / Locust
- L08 腳本設計：參數化、關聯、思考時間、登入態、防快取
- L09 加壓模型：階梯、尖峰、固定併發、容量探測

### M4 執行與監控
- L10 監控體系：APM、資源監控、應用層指標
- L11 全鏈路壓測：影子庫、流量染色、線上壓測
- L12 執行紀律：基線、單變數、暖機、可重現

### M5 瓶頸定位（核心）
- L13 分層排查法：LB → App → DB → 中間件
- L14 DB 瓶頸：慢查詢、鎖、連接池、索引、分庫分表
- L15 App 瓶頸：執行緒池、GC、連接池、N+1
- L16 快取／MQ：Redis 熱 key、穿透／擊穿／雪崩、MQ 堆積

### M6 調優與複盤
- L17 容量規劃：擴容、限流、降級、熔斷
- L18 秒殺專題：庫存扣減、超賣、排隊、預熱
- L19 出報告：寫給 RD/PM 看懂的效能報告
- L20 複盤常態化：CI 整合、效能回歸、看板
