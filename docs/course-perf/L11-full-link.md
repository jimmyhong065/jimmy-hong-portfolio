# L11 全鏈路壓測：影子庫、流量染色、線上壓測

**模組**：M4 執行與監控　|　**狀態**：草稿

## 學習目標

- 理解為何大促要做「線上全鏈路壓測」而非只壓單服務。
- 掌握影子庫/影子表、流量染色、流量隔離三大機制。
- 知道線上壓測的風險控管與資料清理。

## 大綱

1. 開場：單服務壓測綠燈，全鏈路仍可能崩——木桶效應。
2. 為何要線上壓：環境等比難、依賴複雜。
3. 流量染色：標記壓測流量。
4. 影子庫/影子表：壓測寫入隔離。
5. 中間件/快取的影子隔離。
6. 風險控管：限流、熔斷、回滾、清理。
7. 收尾：全鏈路壓測前置條件清單。

## 核心觀念

- **全鏈路壓測**：模擬真實用戶路徑，壓整條依賴鏈（閘道→服務→DB→快取→MQ→第三方），暴露**跨服務的木桶短板**。
- **流量染色**：幫每個壓測請求蓋一個記號（header / 上下文標位）並全鏈路透傳，讓系統一路都認得出「這是壓測流量」。
- **影子庫/影子表**：染色流量的寫入導到影子儲存（同結構、隔離資料），避免污染生產資料。快取、MQ、檔案也要對應影子隔離。
- **線上壓測動機**：生產環境才有真實的規格/拓撲/資料量/依賴；等比測試環境成本高且仍有偏差。
- **風險控管**：壓測開關可秒級關閉、限流閥值、熔斷保護、灰度時段（低峰）、壓後資料清理與核對。

## 實戰／案例

- 木桶效應：訂單服務單壓 OK，但下游庫存服務 + 風控第三方在全鏈路下先飽和，整鏈逾時。
- 染色沒透傳乾淨：某中間件沒識別標記，壓測寫入污染生產報表——上線前要逐跳驗證透傳。
- 線上壓測排在**業務低峰 + 有人值守 + 一鍵熔斷**，並先在預發跑過染色透傳驗證。

## 書本參考

《性能測試實戰30講》ch14（業務模型）：高樓把「生產環境流量複製 / 直接對生產發起壓力」稱為全鏈路壓測，並點出其**最難的不是技術而是組織協調能力**——呼應本課「線上壓測要值守、低峰、一鍵熔斷」。影子庫/染色屬業界實務，書外延伸。
